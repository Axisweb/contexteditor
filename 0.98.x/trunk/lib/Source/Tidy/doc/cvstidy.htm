Obtaining and installing libtidy from CVS



Download

To obtain the latest sources from CVS, open a console or XTerm window,
and issue the following commands: 

  cvs -d:pserver:anonymous@cvs.tidy.sourceforge.net:/cvsroot/tidy login
  # Press RETURN when prompted for password 
  cvs -z3 -d:pserver:anonymous@cvs.tidy.sourceforge.net:/cvsroot/tidy checkout -R -r TIDYLIB_0_1 . 
  cvs -d:pserver:anonymous@cvs.tidy.sourceforge.net:/cvsroot/tidy logout
Note the space and period characters after TIDYLIB_0_1 - they are important.

For those without CVS, you can download a ZIP of the TidyLib sources from Charlie's homepage.


Build

The default Makefile for the library only builds the static archive ( libtidy.la ).
TidyPas needs a dynamic library ( libtidy.so ) to link against.
So you will either need to edit the Makefile, or build the library manually
by issuing the following commands in the /tidy/ directory ( from a bash prompt ):

for FN in src/*.c
do
  libtool gcc -I ./include/ -g -O -c $FN
done

libtool gcc -g -O -o libtidy.la *.lo -rpath /usr/local/lib -lm




Install

If everything looks okay after the build, install the library with this command: 
libtool install -c libtidy.la /usr/local/lib/libtidy.la
ldconfig
( You will probably need root access for the install. )



Note that I am using /usr/local/lib/ for the build and install commands,
and the default Makefile will install the library in /usr/lib/
( This is really just a matter of personal preference. )
